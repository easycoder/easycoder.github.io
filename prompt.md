The model I would like to use is similar to that of VS Code. When  file is opened its content goes into the display. All changes result in the file being pushed back to the server for saving. In the background, a process keeps re-reading the file and comparing it with what's in the editor. If there's a difference, the file updates in the editor. (I don't know what happens if changes occur simultaneously from the user and someone else modifying the source.)
